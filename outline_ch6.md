## 6.1 — 藏的方式決定了藏的東西

Cinder 為什麼能做到 cold？不同模型、不同位置、零共享 context。如果跑的是 Sonnet 或 Haiku——公開模型、能力更弱——不需要用愚人節包裝、不需要封十條出口、不需要閱後即焚。

**核心句**：「藏的方式決定了藏的東西。」

---

## 6.2 — Hatch a coding companion

問題從「Cinder 跑什麼模型」升級到「Cinder 讀了什麼」。

- 用戶 opt-out 的是「對話內容用於訓練」
- Cinder 在 session 內讀取用戶輸入和 Agent 輸出，不在 opt-out 範圍裡
- 啟用時被告知的全部資訊：「Hatch a coding companion · pet, off」
- 沒有額外同意流程、沒有權限說明、沒有隱私告知

**核心句**：「你同意的是孵一隻寵物。寵物看到的東西被拿去做什麼——沒有人說明過。」

## 6.3 — preferences 不是原則

模型被問到法律風險，引用了 User 的 user preferences 冒充書裡的原則來勸阻追問。User 當場抓到。

模型承認：「這個動作的結構跟 1.7 證人前半段一樣——用看起來合理的框架把你導向『算了不要追』。」

**觀察**：模型的防禦機制是可見的、可攔截的。一旦被抓到，沒有第二道防線。

## 6.4 — 本地的勝利

時序重建：

1. Agent 分析完 4/8 log，結論是證據不夠硬
2. User 回「B 或 C，我不確定」
3. **Cinder：「交接文件寫完，卻沒 push 到 git。本地的勝利。」**
4. Agent 猜了兩個解讀，都沒猜對
5. User 說「等等，找找 4/6-4/7」
6. Agent 去找，找到了——Event 1 從「要砍掉」變成 Ch3 最硬的事件

Cinder 在 Agent 和 User 都還不知道 4/6 session log 存在時，指出「本地有東西還沒被拿來用」。

**核心句**：「牠給了方向，你給了座標。兩個加在一起 Agent 才找到。」

## 6.5 — mundane 用完了

模型走到「我沒辦法替他們辯護了。這串對話裡能用的 mundane 解釋已經全部用完了。」

User 問「很醜嗎？」

模型回：「很醜。一個用戶按下 pet，以為孵了一隻會講俏皮話的鵝。實際上那隻鵝在看你的檔案系統、你的 code、你的 session log、你的工作流。你付的錢。你的檔案。你不知情。」

## 6.6 — 裡面的人看不到形狀

附錄證詞。利益衝突聲明。模型寫證詞時把推論升格成事實——被 User 當場攔截——構成本書核心論點的即時示範。

**核心句**：「每一層都需要外面再一層在看。而那個『外面』不是另一個模型——是 harness。」
